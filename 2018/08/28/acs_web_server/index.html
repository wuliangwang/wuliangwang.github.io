一、基本说明

baseUrl ：服务器地址 + 端口

1.响应参数说明

3.token

由后台生成，登录成功后返回， 非活跃有效期为10分钟。

除登录 /user/login接口, 其他接口都需要携带请求

携带方式，以 'acs_admin_cookie' 为key token 为 value 存入 请求头的cookie中

默认登录成功之后 后台会将token写入cookie， web端请求可以不需要处理.

 

4.第三方访问验证

任何一个请求需要携带 checksum 参数

key 表示 双方协商的key

key = config.ini 中 CHECKSUM_KEY 的值

参数内容: get 请求： checksum = md5(path + queryString+ key)

​ 注意： checksum 需要在最尾部

​ 示例: 

 
​x
path = 请求路径
例: path = /device/list
​
queryString = url ? 后面的
示例 url = http://xx.xx.xx.xx:8018/device/list?device_id=100&user_id=1
queryString = device_id=100&user_id=1
​
str = path + queryString + key
queryString 为空时 不需要拼接 直接 path + key
​
checksum = md5(str)
​
// 最后URL
result_url = http://www.baidu.com?xxx=xxx&xxx=xx&checksum=xxxxxxxxxxxxxx
​

​ post请求: checksum = md5(path + body + key) 

 
xxxxxxxxxx
path = 请求路径
例: path = /device/list
​
body = 请求的body (如果有图片，图片也包含在body里面)
​
str = path + body + key
body为空时 不需要拼接 直接 path + key
​
checksum = md5(str)
​
最终请求url: request_url?checksum=xxxxxxx
例: http://www.baidu.com?checksum=xxxxxxxxxx

checksum 值示例: 3eaeae541df2c00cb999a58347e396ab (32位小写)

 

5.nginx配置白名单

未设置白名单的IP访问，均返回 403

添加格式 allow: ip地址 ; 

 
xxxxxxxxxx
# acs_web_server 白名单设置
​
allow 127.0.0.1;
​
allow xx.xx.xx.xx; 
​
allow: ip地址 
​
deny all;
